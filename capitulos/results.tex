\chapter{Estado atual da pesquisa}
\label{cap:resultados-preliminares}
Após a execução do estudo empírico com novatos em software livre, algoritmos de classificação começaram a ser avaliados seguindo as condições estabelecidas na Seção \ref{sec:classificacao}. Preliminarmente, os resultados apresentados pelos modelos de classificação gerados ainda não são significativos se comparados aos apresentados pelo modelo de \citet{prana2019categorizing}, que também buscam identificar um conjunto de categorias em documentações de projetos em software livre. O valor médio da métrica de avaliação \textit{f-measure} obtido pelos autores do trabalho relacionado para o algoritmo de classificação \textit{Random Forest}, por exemplo, é de $0,696$, enquanto que o valor obtido para este estudo para o mesmo algoritmo é de $0,389$. Todos os algoritmos propostos na etapa de treinamento já foram testados, e nenhum deles até o momento obteve \textit{f-measure} maior que $0,5$.

Acreditamos que a razão para os baixos resultados dos algoritmos de classificação, se concentra no fato de que os dados do estudo empírico ainda não estão sendo pré-processados corretamente, ou não são suficientes para predição de tais categorias. No total, 4983 parágrafos foram analisados pelos alunos, e as sete categorias foram anotadas 3432 vezes. Para avaliar se os parágrafos analisados pelos alunos convergiam em uma mesma anotação, calculamos a concordância estatística entre pares de alunos que avaliaram a documentação de um mesmo projeto de software livre, utilizando o método estatístico Fuzzy Kappa para cálculo da concordância \citep{kirilenko2016fuzzy}. No estudo proposto por \citet{prana2019categorizing}, parágrafos de documentação foram analisados por dois avaliadores, um aluno de doutorado e um desenvolvedor da indústria, e o mesmo cálculo de concordância foi aplicado. A concordância obtida no trabalho relacionado foi de $0,858$, enquanto que a média das concordâncias dos alunos neste estudo foi de $0,188$, substancialmente mais baixa que a apresentada pelos autores relacionados. 

Acreditamos que a razão para baixa concordância se concentra no modo como o estudo empírico foi aplicado. Muito provavelmente, os alunos não tiveram tempo suficiente para compreender as categorias e realizar a análise qualitativa corretamente, gerando discordância nas marcações feitas por eles. Por conta da baixa performance dos algoritmos de classificação e a baixa concordância apresentada pelos alunos no estudo empírico, pretendemos realizar uma nova etapa de identificação das categorias antes de construir o modelo de classificação. Esta nova etapa será realizada pelos próprios pesquisadores que estão conduzindo este estudo, e ela não só contribuirá para incrementar o número de parágrafos marcados, como também servirá para compreender se, de fato, a análise dos alunos não foi executada como deveria. Os códigos implementados preliminarmente para construção do modelo de classificação, e todos os dados e ferramentas utilizadas nesta pesquisa, já se encontram disponíveis em um repositório de código\footnote{\url{github.com/fronchetti/usp}}.
