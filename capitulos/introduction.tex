Projetos de software livre estão presentes em várias áreas do desenvolvimento de software, atendendo a uma parcela significava do mercado consumidor de programas de computador~\citep{bonaccorsi2003open}. Por funcionarem de modo diferente aos projetos proprietários, estes necessitam de contribuições voluntárias para se manterem ativos~\citep{crowston2012free}. Voluntários, ao tentarem ingressar em projetos de software livre, costumam sofrer uma série de dificuldades, que comprometem a realização de contribuições~\citep{steinmacher2014thehard}. Tais dificuldades levam, muitas das vezes, novatos a desistência de contribuir com projetos de software livre, refletindo no enfraquecimento das comunidades, que necessitam de novas contribuições. 

Entre as dificuldades enfrentadas por novatos, se encontram barreiras associados as documentações dos projetos de software livre que, muitas das vezes, se encontram obsoletas, incompletas, incoerentes ou inconsistentes a realidade do sistema desenvolvido~\citep{briand2003enough, aghajani2019issues}. Além do mais, evidências mostram que em muitos casos, as documentações de projetos de software livre, não atendem as necessidades que novos contribuidores tem~\citep{steinmacher2014thehard}. Tais problemas de documentação, tendem a gerar confusão naqueles que desejam realizar uma primeira contribuição, e por esta razão, precisam ser solucionados. 

Para minimizar problemas de documentações de software, uma série de ferramentas são propostas por cientistas e desenvolvedores. \citet{zhong2013detecting}, por exemplo, sugerem um mecanismo de identificação de erros em documentações, que utiliza uma combinação de técnicas de processamento de linguagem natural, para encontrar em códigos fonte, trechos de documentação que estejam incorretos ou obsoletos. Outro exemplo de trabalho é proposto por \citet{haiduc2010automated}, que estabelece um mecanismo de geração automática de documentação para classes e métodos em Java, visando aprimorar a compreensão de códigos fonte. Até mesmo fora do contexto acadêmico, uma série de projetos relacionados a documentações de software também são propostos, entre eles, o Javadoc, que analisa anotações e comentários de código, e gera automaticamente páginas de documentação para os códigos analisados em formato HTML.  

Apesar dos trabalhos apresentados contribuírem com documentações de software, nenhuma das ferramentas propostas busca contribuir com documentações para novatos em software livre, tampouco contribuem com a avaliação das documentações disponibilizadas por comunidades neste contexto. Por esta razão, é proposto para este estudo, a criação de um modelo de classificação que identifique trechos em documentações de projetos de software livre que sejam relevantes a novos contribuidores. Este modelo de classificação deverá não só auxiliar novatos a encontrarem documentações apropriadas para quem almeja contribuir com software livre, como também podem ser úteis para avaliar a qualidade das documentações já existentes em projetos de software livre.

\section{Estudo preliminar}
Esta estudo dá sequência a uma pesquisa preliminar \citep{fronchetti2019attracts}, onde foram investigadas características que influenciam a entrada de novatos em projetos de software livre. Nesta pesquisa preliminar, 15 características de 450 projetos de software livre foram extraídas e analisadas, entre elas, a idade, o domínio de aplicação e o número de linguagens de programação utilizadas em cada um dos projetos selecionados. O algoritmo de clusterização K-Spectral Centroid (KSC) foi utilizado para investigar a taxa de crescimento de novatos nos projetos, e três diferentes padrões de entrada de novatos foram encontrados: logarítmico, exponencial e linear. 


Com base nesses padrões de crescimento, um modelo de classificação foi construído a fim de compreender quais fatores poderiam explicar as taxas de entrada de novatos em cada um dos projetos. Foi constatado que fatores como a popularidade dos projetos (representada pelo número de estrelas), o tempo para revisão de submissões, a idade dos projetos, e as linguagens de programação utilizadas nós repositórios de código, eram os fatores que melhor explicavam a entrada de novatos em projetos de software livre.

Entre as características avaliadas, também foram analisados fatores dicotômicos associados a documentação dos projetos, tais como a existência de arquivos README e CONTRIBUTING nos repositórios de código, conhecidos por serem fontes primárias de informação para novatos em software livre. No entanto, as características relacionadas a documentação dos projetos não apresentaram grande influência na atratividade de novatos (A existência de um código de conduta, por exemplo, foi o fator menos influente entre os avaliados). 


\section{Questões de pesquisa}
\label{sec:questoes-de-pesquisa}
A metodologia proposta para este estudo é fundamentada a partir de três questões pesquisa, focadas em compreender e avaliar o modelo de classificação a ser construído, cujo objetivo é identificar em trechos de documentações, pontos que sejam relevantes a novos contribuidores:

\textbf{QP1} É possível identificar trechos em documentações de projetos de software livre que sejam relevantes a novos contribuidores utilizando algoritmos de classificação?

\textbf{QP2} Como diferentes características influenciam  as predições feitas pelo classificador de trechos de documentação?

\textbf{QP3} Qual a percepção dos desenvolvedores no contexto de software livre quanto a identificação de trechos de documentação relevantes a novos contribuidores?
